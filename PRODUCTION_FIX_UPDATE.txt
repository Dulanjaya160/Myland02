WHAT WAS CHANGED:
═════════════════

Function: handleProductionSubmit()

Before (Wrong):
───────────────
usedIngredients: [{
    ingredient: { id: parseInt(ingredientIds[i]) },
    quantity: parseFloat(ingredientQuantities[i])
}]

After (Correct):
────────────────
usedIngredients: [{
    ingredientId: parseInt(ingredientIds[i]),
    quantityUsed: parseFloat(ingredientQuantities[i])
}]

This matches what the backend expects (ProductionIngredientRequest DTO).

TECHNICAL DETAILS:
══════════════════

The backend ProductionController expects:
- ProductionRequest with usedIngredients array
- Each item should be ProductionIngredientRequest with:
  * ingredientId (Long)
  * quantityUsed (double)

The frontend was incorrectly sending:
- ingredient: { id: ... }
- quantity: ...

This caused the backend to receive null values, triggering:
"IllegalArgumentException: The given id must not be null"
