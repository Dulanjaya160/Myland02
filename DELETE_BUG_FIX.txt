ISSUE FIXED:
════════════
After deleting production records (or other data), you couldn't add new records.

ROOT CAUSE:
═══════════
1. Detached Entity Problem: The Product entity from the request wasn't attached
   to the current Hibernate session, causing "detached entity" errors.

2. SQLite Auto-Increment: After deletions, SQLite's sequence could get out of sync.

3. Missing Foreign Key Constraints: SQLite wasn't enforcing foreign keys properly.

══════════════

1. ProductionController.java

✓ Added detailed logging for debugging

TECHNICAL DETAILS:
══════════════════

The Problem:
────────────
When you submit a production form, the frontend sends:
```json
{
  "product": { "id": 1 },
  "date": "2026-02-08",
  "producedUnits": 100,
  "usedIngredients": [...]
}
```

The Product object { "id": 1 } is just a reference, not a managed entity.
Hibernate tried to persist it as a new entity, causing conflicts.

The Solution:
─────────────
Now the controller does:
```java
Product product = productRepo.findById(request.getProduct().getId())
    .orElseThrow(() -> new RuntimeException("Product not found"));
production.setProduct(product);
```

This fetches the actual managed Product entity from the database.
