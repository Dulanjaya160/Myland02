WHY 300ms DELAY?
════════════════
SQLite with Hibernate needs time to:
1. Commit the transaction to disk
2. Release database locks
3. Make data available for subsequent reads

The 300ms delay ensures the transaction fully completes before we query.
This is especially important for SQLite which uses file-based locking.


TECHNICAL DETAILS:
══════════════════

The issue was a race condition between:
1. Database WRITE (save operation)
2. Database READ (load operation)

Without delay:
- Save starts → Returns success → Load starts immediately
- But SQLite transaction not yet committed
- Load returns old/empty data

With delay:
- Save starts → Returns success → Wait 300ms → Load starts
- SQLite transaction committed and flushed
- Load returns fresh data
